## 1. 벤치마크 분석

### 1.1 무신사 (musinsa.com) — 직접 확인 완료

**확인 URL**: `musinsa.com/category/001/goods` (상의 카테고리)

**필터 구조 (3-Layer 시스템):**

| 레이어 | 위치 | 구성 | 동작 |
|--------|------|------|------|
| ① 서브카테고리 탭 | 상단 고정 | 수평 스크롤 텍스트 탭 (전체, 반소매 티셔츠, 셔츠/블라우스, 긴소매 티셔츠 등) | 클릭 시 카테고리 전환, URL 변경 |
| ② 드롭다운 필터 바 | 탭 아래 고정 | 수평 필터 버튼 (성별, 컬러, 패턴/무늬, 소재, 스타일, 가격, 사이즈, 핏, 브랜드, 혜택, 상세옵션, 상품유형) | 각 버튼 클릭 → 드롭다운 패널 확장 |
| ③ 상세필터 모달 | 풀스크린 오버레이 | 탭형 네비게이션 (추천·성별·컬러·패턴/무늬·소재·스타일·가격·사이즈·핏·브랜드·혜택) + 체크박스 그룹 | "상세필터열기" 클릭 → 모달 오픈, 하단 CTA "N개의 상품보기" |

**핵심 UX 패턴:**
- 필터 선택 시 즉시 결과 갱신 (모달 내부에서는 CTA로 일괄 적용)
- 결과 수 실시간 표시 ("345,581개")
- 정렬: "무신사 추천순" 드롭다운
- 모바일: 동일 구조, 필터 바가 수평 스크롤로 축소

**Packlinx에 참고할 점 (구조/배치만, 비주얼 스타일 제외):**
- 서브카테고리 + 패싯 필터를 분리하는 2-레이어 구조
- 필터별 독립 드롭다운 패널 (한 번에 하나만 열림)
- 결과 수를 CTA 버튼에 포함하여 행동 유도
- 모바일에서 바텀시트/모달 전환

### 1.2 Alibaba.com — 직접 확인 완료 (v2 추가)

**선정 근거:** 글로벌 B2B 1위 플랫폼. Packlinx와 동일한 B2B 공급업체 디렉토리 성격이며, 대규모 패싯 필터 UX의 실전 사례를 확인할 수 있음.

**확인 URL**: `alibaba.com/trade/search?searchText=packaging+box`

**필터 구조 (좌측 사이드바 + 상단 탭):**

| 구성 요소 | 위치 | 동작 |
|-----------|------|------|
| 상단 탭 | 헤더 아래 | 제품 / 공급업체 / 전 세계 (검색 범위 전환) |
| 좌측 사이드바 (~280px) | 콘텐츠 영역 왼쪽 | 8+ 아코디언 섹션: 필터(Guaranteed/Trade Assurance), 공급업체 특징(Verified/PRO), 배송 예정일, 스토어 리뷰(별점), 제품 특징(샘플), 범주(카테고리 링크), 가격(Min-Max 입력), 최소 주문량(텍스트 입력), 공급 국가/지역 |
| 결과 수 + 정렬 | 콘텐츠 상단 | "100,000+개" 표시, 관련 높은 순 드롭다운, 그리드/리스트 뷰 토글 |
| 제품 카드 | 리스트 뷰 | 대형 썸네일 + 타이틀 + 재주문율 + 가격 범위 + MOQ + 판매량 + 공급업체명 + 연수/국가/평점 |

**핵심 UX 패턴:**
- 필터 타입 다양성: 체크박스, 라디오 버튼, 텍스트 입력(가격/MOQ), 텍스트 링크(범주)
- 각 섹션 볼드 헤더 + 구분선으로 시각적 분리
- B2B 특화 필터: Verified 업체, Trade Assurance, MOQ, 재주문율
- 사이드바 내 독립 스크롤 (콘텐츠 영역과 별도)

**Packlinx에 참고할 점 (구조/배치만, 비주얼 스타일 제외):**
- B2B 디렉토리에서는 **신뢰 기반 필터**(인증, 검증 업체)가 최상단에 배치됨
- 가격/MOQ 같은 범위 필터는 텍스트 입력 방식 — Packlinx의 현재 패싯은 선택형이므로 당장 적용 대상 아님
- 결과 수를 필터 영역이 아닌 콘텐츠 상단에 배치하여 필터 ↔ 결과 관계를 명확히 함

### 1.3 지그재그 (zigzag.kr) — 직접 확인 완료 (v2 추가)

**선정 근거:** 한국 여성 패션 버티컬 플랫폼으로, 고도화된 모바일 퍼스트 필터 UX를 선보임. Packlinx의 모바일 바텀시트 설계에 참고할 패턴이 풍부함.

**확인 URL**: `zigzag.kr/categories/-1?title=전체`

**필터 구조 (모바일 퍼스트 수평 필터):**

| 구성 요소 | 위치 | 동작 |
|-----------|------|------|
| 수평 카테고리 탭 | 헤더 아래 | 전체, 상의, 아우터, 팬츠, 원피스 등 — 탭 클릭 시 카테고리 전환 |
| 인라인 필터 체크박스 | 탭 아래 | 직진배송 □, 빠른출발 □ — 간단한 boolean 필터 |
| 정렬 드롭다운 | 우측 상단 | 직짹추천순 ▾ |
| 제품 그리드 | 3열 그리드 | 브랜드명 + 상품명 + 할인율/가격 + 무료배송 태그 + 별점/리뷰 수 |

**핵심 UX 패턴:**
- 극도로 미니멀한 필터 — boolean 체크박스 2개만 인라인 노출
- 카테고리 전환은 탭으로, 패싯 필터는 최소화
- 모바일 최적화: 필터가 콘텐츠 공간을 거의 차지하지 않음
- 하단 고정 네비게이션 바 (홈, 카테고리, 찜, 마이페이지)

**Packlinx에 참고할 점 (구조/배치만, 비주얼 스타일 제외):**
- 패싯이 적을 때(2~3개)는 인라인 체크박스가 가장 효율적
- 모바일에서 필터 영역을 최소화하면 콘텐츠 도달 속도가 빨라짐
- 그러나 Packlinx는 패싯 6종으로 인라인에는 과다 → 드롭다운 또는 바텀시트 필요

### 1.4 G마켓 (gmarket.co.kr) — 직접 확인 완료 (v2 추가)

**선정 근거:** 한국 대형 오픈마켓으로, 최근 리디자인에서 모바일 퍼스트 검색 결과 UI를 도입. 전통 좌측 사이드바를 폐지한 사례로서 필터 트렌드 변화를 보여줌.

**확인 URL**: `gmarket.co.kr/n/search?keyword=포장박스`

**필터 구조 (리디자인 후):**

| 구성 요소 | 위치 | 동작 |
|-----------|------|------|
| 검색 바 | 상단 고정 | 키워드 검색, 스티키 |
| 프로모션 섹션 | 검색 결과 상단 | "오늘의 프라임상품", "먼저 둘러보세요", "인기 상품이에요" 순차 노출 |
| 상품 카드 | 리스트 뷰 | 대형 썸네일 + 상품명 + 가격/할인 + 멤버 적립 + 배송 + 별점/구매수 |
| 좌측 사이드바 | **없음** (리디자인 후 제거) | — |

**핵심 UX 패턴:**
- 전통적 좌측 사이드바 필터를 완전히 제거하고 프로모션 중심 UX로 전환
- 검색 키워드 기반 결과 → 필터 없이 추천 알고리즘 중심
- 모바일과 데스크톱 동일한 단일 컬럼 레이아웃

**Packlinx에 참고할 점:**
- 대형 오픈마켓도 사이드바 필터를 제거하는 추세 — 그러나 이는 *상품* 검색 UX의 트렌드이며, *업체 디렉토리*에는 다른 기준 적용 필요
- 결과가 수백만 개인 마켓플레이스와 달리 Packlinx (1,380개 업체)는 필터가 정밀 탐색에 핵심
- **시사점:** Packlinx는 G마켓 방향(필터 제거)이 아닌, 무신사/Alibaba 방향(컴팩트 필터 유지)이 맞음

## 2. 현황 분석

### 2.1 현재 카테고리 페이지 구조

**파일**: `src/app/categories/[slug]/page.tsx`

```
┌──────────────────────────────────────────────┐
│ 헤더 (nav, 로고)            #0F172A 배경       │
├──────────────────────────────────────────────┤
│ 카테고리 히어로 (아이콘 + 이름 + 업체 수)      │
├──────────────────────────────────────────────┤
│ ▼ 스티키 필터 바 (수직 적층)                   │
│   [소재 칩 행]                                │
│   [용도 칩 행]                                │
│   [형태 칩 행 — PackagingFormFilter]           │
│   [인증 아코디언 — CertFilterAccordion]        │
│   [샘플 가능 토글]                             │
├──────────────────────────────────────────────┤
│ 활성 필터 칩 표시 (✕ 제거)                     │
├──────────────────────────────────────────────┤
│ 업체 그리드 (1→2→3열 반응형)                   │
│ PAGE_SIZE=30, 페이지네이션                     │
├──────────────────────────────────────────────┤
│ 관련 블로그 / 관련 카테고리 / 기타 카테고리    │
├──────────────────────────────────────────────┤
│ 풋터                                          │
└──────────────────────────────────────────────┘
```

### 2.2 기존 필터 패싯 (6종)

| # | 패싯 | DB 컬럼 | 타입 | UI 형태 | 다중선택 |
|---|------|---------|------|---------|---------|
| 1 | 소재 (material) | `material_type` | text | 수평 칩 행 | ✅ |
| 2 | 형태 (form) | `packaging_form` | text | 수평 칩 + 더보기 | ✅ |
| 3 | 인증 (cert) | `certifications` | text[] | 아코디언 체크박스 | ✅ |
| 4 | 용도 (use-case) | use_case_tags 테이블 | relation | 수평 칩 행 | ❌ (단일) |
| 5 | 샘플 (sample) | `sample_available` | boolean | 토글 버튼 | N/A |
| 6 | 유형 (subtype) | `subcategory` | text | 수평 칩 행 (인쇄 카테고리만) | ❌ (단일) |

**쿼리 파라미터**: `?material=X&form=Y&cert=Z&use-case=W&sort=S&sample=true&page=N`

### 2.3 한계점

| 문제 | 설명 | 영향 |
|------|------|------|
| **수직 적층으로 인한 높은 필터 영역** | 6개 패싯이 세로로 쌓여 첫 화면에서 업체 카드가 밀려남 | 모바일에서 특히 심각 — 필터만으로 스크롤 2회 이상 필요 |
| **패싯 간 시각적 구분 부재** | 모든 칩이 유사한 스타일로 나열되어 어떤 필터인지 빠르게 구분 어려움 | 전문 바이어의 필터 탐색 속도 저하 |
| **필터 적용 결과 수 미표시** | 필터 변경 시 총 결과 수가 눈에 띄지 않음 | 사용자가 "이 필터가 결과를 줄이는지" 직관적 판단 불가 |
| **모바일 UX 부재** | 데스크톱과 동일한 수직 칩 나열 → 화면 대부분 필터가 차지 | 360px에서 사용성 심각히 저하 |
| **일부 필터 색상 불일치** | 형태 필터 `#7C3AED` 퍼플, 소재 필터 `#C2410C` 브라운 — 브랜드 색상 시스템과 불일치 | 시각적 일관성 저하 |

## 3. 권고안 (Variant A): 상단 드롭다운 필터 바

### 3.3 인터랙션 흐름

```
[필터 버튼 클릭]
    │
    ├── 데스크톱: 해당 패싯 드롭다운 패널 펼침
    │     ├── 체크박스 선택/해제 → 즉시 URL 반영 + 결과 갱신
```

## 6. 선택 추천

### 권고안 (A: 상단 드롭다운 필터 바)를 추천합니다.

**근거**: Packlinx는 현재 업체 수 1,380개, 필터 패싯 6종으로 **중규모 디렉토리**입니다. 이 규모에서 좌측 사이드바는 시각적으로 과잉 — 사이드바 콘텐츠 양 대비 차지하는 공간이 비효율적입니다. 반면 드롭다운 필터 바는:

1. **콘텐츠 영역 전체 너비 보존** → 업체 카드 정보 밀도 유지 → 전문 바이어가 한 화면에서 더 많은 업체를 비교 가능
2. **모바일 전환이 자연스러움** → 바텀시트는 한국 모바일 사용자가 가장 익숙한 패턴 (배민, 무신사, 당근 등)
3. **구현 공수 40% 절감** → 기존 스티키 바 구조를 활용하므로 레이아웃 재설계 불필요
4. **DAU 300 목표와의 연결** → 필터 탐색 속도 향상으로 카테고리 페이지 체류 시간↑ 재방문율↑

사이드바(B안)는 업체 수가 10,000+ 이상으로 성장하고 필터 패싯이 10종 이상으로 확대될 때 재검토하는 것이 적절합니다.

### 9.2 핵심 발견 사항

1. **한국 커머스의 필터 UI 양극화:**
   - 대형 마켓플레이스(G마켓)는 필터를 제거하고 추천 알고리즘 중심으로 이동
   - 버티컬 플랫폼(무신사, 지그재그)은 카테고리 탭 + 컴팩트 필터를 유지
   - B2B 디렉토리(Alibaba)는 풍부한 사이드바 필터를 여전히 사용

2. **Packlinx 맥락에서의 시사점:**
   - 업체 수 ~1,380개: G마켓 방향(필터 제거)은 부적합, 필터가 정밀 탐색에 필수
   - 패싯 6종: 지그재그 방향(인라인 2개)에는 과다, 무신사 방향(드롭다운 바)이 적합
   - B2B 특성: Alibaba의 신뢰 필터(인증) 최상단 배치 패턴 적용 가치 있음

3. **권고안 A 보강 포인트 (추가 벤치마크 반영):**
   - Alibaba에서 차용: 인증 필터를 드롭다운 바에서 좌측(우선순위 높은 위치)에 배치
   - 지그재그에서 차용: 모바일에서 필터 영역을 최소화하는 "필터 버튼 1개 + 바텀시트" 패턴 유지 (이미 권고안 A에 포함)
   - G마켓 교훈: 결과 수 표시를 반드시 포함 — 필터의 효과를 사용자가 즉시 인식해야 함
   - 무신사 패턴 재확인: 드롭다운 필터 바가 한국 사용자에게 가장 익숙한 패턴

### 9.3 권고안 A 수정 사항 (v2)

v1 권고안 A의 핵심 구조(수평 드롭다운 필터 바)는 추가 벤치마크 분석 후에도 유효합니다. 다음 사항만 보강:

| 항목 | v1 | v2 보강 |
|------|-----|--------|
| 필터 버튼 순서 | 소재 → 형태 → 인증 → 용도 → 샘플 | **인증 → 소재 → 형태 → 용도 → 샘플** (B2B에서 인증이 구매 결정에 가장 큰 영향, Alibaba 패턴 참고) |
| 결과 수 표시 | "결과 N개" 텍스트 | **"N개 업체"** 로 변경 — "업체"를 명시하여 상품이 아닌 업체 디렉토리임을 강조 |
| 모바일 바텀시트 | 전체 패싯 스크롤 | **아코디언 기본 닫힘** — 지그재그의 미니멀 패턴 참고, 첫 화면에 CTA 버튼이 보이도록 |
| 필터 버튼 내 카운트 | 없음 | **활성 필터 수 뱃지** — 소재(2) 형태(1) 처럼 선택된 필터 수를 버튼 내에 표시 |
